# MetaSwap — демонстрационный обменник TON ↔ USDT

MetaSwap — упрощённый веб-интерфейс обменника, который имитирует подключение популярных кошельков
(TonKeeper, Trust Wallet, MetaMask) и полный цикл операции TON ↔ USDT. Проект собран как статический
фронтенд (HTML/CSS/JS) и может использоваться для презентаций, согласования требований и дальнейшей
разработки бэкенда.

## Что реализовано

- **Подключение кошелька** — модальное окно с выбором TonKeeper, Trust Wallet или MetaMask, генерация
  тестового адреса и отображение статуса сессии в шапке.
- **Дашборд обмена** — единая карточка со сменой направления TON ↔ USDT, пересчётом курсов, комиссий и
  доступного баланса в реальном времени.
- **Логика swap** — списание средств с кошелька, зачисление результата на баланс MetaSwap, обновление
  объёма за 24 часа и счётчика сделок.
- **Вывод TON** — отдельная кнопка, которая возвращает накопленный TON обратно на подключённый кошелёк.
- **Метрики** — курс, комиссия, объём и активность за последний час синхронизируются между панелью возле
  формы и блоком показателей ниже.
- **Поддержка** — быстрые ссылки на Telegram и почту, а также юридический адрес в футере.

## Локальный просмотр

1. Откройте терминал в корне репозитория и выполните любой из вариантов запуска статического сервера:
   ```bash
   npx serve MetaRefund
   # или
   python3 -m http.server --directory MetaRefund 4173
   ```
2. Перейдите по адресу `http://localhost:3000` или указанному в выводе порту.
3. Авторизуйте один из кошельков, задайте сумму обмена и убедитесь, что статус операции обновляется.

> ⚠️ Все операции выполняются локально в браузере и не взаимодействуют с реальной сетью TON.

## Развёртывание

- **GitHub Pages** — загрузите папку `MetaRefund` в репозиторий и включите Pages (`Deploy from a branch`,
  ветка `main`, папка `/`).
- **Vercel / Netlify** — импортируйте репозиторий и укажите `MetaRefund` как корневую директорию билда.
- **S3/CloudFront** — загрузите содержимое папки в бакет и подключите CDN для производственного использования.

## Следующие шаги для продакшена

1. Подключить реальные SDK TonKeeper и WalletConnect v2, реализовать подпись для авторизации.
2. Развернуть backend (FastAPI или NestJS) с PostgreSQL, Redis и очередями для отслеживания транзакций.
3. Интегрировать провайдера KYC/AML, управление комиссиями и вебхуки для on-chain событий.
4. Добавить реальные операции swap/withdraw через DEX (STON.fi) или внутренний маркет-мейкер.
5. Настроить CI/CD, мониторинг (Prometheus, Grafana) и безопасное хранение ключей (Vault/KMS).

Демо-приложение поможет согласовать UX и требования прежде, чем переходить к реализации продакшен-стека.
